# 面向异常恢复的 Sim-Real 双源经验库与沙盒推演研究方案

## 1. 研究背景

在具身智能与机器人操作任务中，仿真环境常被用于策略训练、异常复现和低成本试错。然而，仿真策略迁移到真实机器人时通常会遇到 Sim-to-Real Gap。该差异主要来自接触动力学、摩擦参数、物体质量、传感器噪声、视觉遮挡、硬件漂移以及真实环境中的长尾异常。

对于真实机器人而言，异常发生后的直接试错成本较高，可能导致任务失败、物体损坏或机器人硬件风险。相反，如果仅依赖仿真中的异常恢复经验，由于仿真与真实环境之间存在偏差，仿真中有效的恢复动作未必能在真机上成功执行。

因此，本研究希望构建一种面向异常恢复的双源经验库，分别存储仿真运行中的异常处理经验与真机运行中的异常处理经验，并利用两类经验之间的差异信息，在真机执行恢复动作之前进行仿真沙盒推演，使机器人能够先在内部模拟环境中“思考”和验证，再将恢复策略下发到真实硬件。

## 2. 核心研究目标

本研究的核心目标是构建一个 Sim-Real 双源经验库与沙盒推演决策框架，使真实机器人在遇到异常时，能够利用历史仿真经验和真机经验完成更可靠的异常恢复。

可以将研究问题概括为：

**仿真异常处理经验与真机异常处理经验能否共同帮助机器人弥合 Sim-to-Real Gap，并提升真实环境下的异常恢复成功率？**

具体目标包括：

1. 构建包含仿真异常经验和真机异常经验的双源经验库。
2. 设计统一的异常经验表示格式，记录异常状态、恢复动作、执行结果和 Sim-Real 差异。
3. 设计基于经验检索的异常恢复策略生成方法。
4. 构建真机执行前的仿真沙盒推演机制。
5. 通过机器人操作任务验证双源经验库对异常恢复成功率和安全性的提升。

## 3. 总体思路

本方案的基本思想是：当真机在执行任务过程中检测到异常时，系统不立即在真实环境中尝试恢复动作，而是先从经验库中检索相似案例，包括仿真经验和真机经验。随后，系统根据真机经验估计仿真与现实之间的差异，并对仿真沙盒进行参数或状态修正。机器人在修正后的仿真环境中推演多个候选恢复策略，选择成功率高、风险低的策略，再下发给真机执行。

整体流程如下：

```text
真机异常检测
→ 检索真机相似经验
→ 检索仿真相似经验
→ 估计 Sim-Real 差异
→ 修正仿真沙盒
→ 推演候选恢复动作
→ 选择最优恢复策略
→ 真机执行
→ 执行结果回写经验库
```

该流程形成一个闭环：真机异常经验不断补充经验库，经验库又不断改善后续异常发生时的沙盒推演效果。

结合当前文件夹中的相关论文，本方案不宜扩展为过大的通用机器人记忆系统，而应聚焦为一个异常恢复闭环。现有论文对本方案的支撑关系如下：

1. `Worth Remembering1.pdf` 支撑经验筛选机制，即不是所有轨迹都进入长期经验库，而是优先保存惊喜度高、异常性强、未来复用价值高的片段。
2. `Learning From Failure.pdf` 支撑失败经验的显式保存与预警，即失败轨迹不是噪声，而是异常恢复系统中最有价值的经验来源之一。
3. `Reconciling Reality through Simulation3.pdf` 支撑 Real-to-Sim-to-Real 思路，即先把真实场景转入仿真，再在仿真中训练或验证策略，最后回到真机执行。
4. `Plan in Sandbox1.pdf` 支撑沙盒推演思想，即机器人可以在物理约束下的内部空间中先推演再执行。
5. `RoboMemory.pdf`、`RoboMME.pdf` 和 `RoboMemArena.pdf` 支撑机器人记忆需要区分时间、空间、对象和程序性经验，而不是只保存扁平文本。
6. `RAPv1.pdf` 和 `Evolvable Embodied Agent for Robotic.pdf` 支撑基于经验检索和反思的规划更新。
7. `RoboCritics1.pdf` 支撑对候选轨迹进行安全约束检查和评分。
8. `Robotic Sim-to-Real Transfer for Long-Horizon Pick-and-Place Tasks in.pdf` 支撑实验场景应优先选择长时程 pick-and-place，并重点关注感知误差和执行误差。

因此，本研究的核心应进一步收敛为：

**面向长时程机器人操作异常恢复，构建一个能够筛选、检索、校正和复用 Sim-Real 双源异常经验的沙盒推演系统。**

## 4. 双源经验库设计

### 4.1 仿真异常处理经验库

仿真异常处理经验库主要记录机器人在仿真环境中遇到的异常、恢复尝试以及执行结果。由于仿真环境成本低、可重复性强，可以系统性地生成大量异常场景，用于覆盖不同物体、姿态、接触状态和扰动条件。

仿真经验可包含以下内容：

1. 任务类型，例如抓取、放置、推动、插入等。
2. 异常类型，例如抓取失败、物体滑落、碰撞、目标位姿偏移、遮挡、夹爪闭合失败等。
3. 异常发生前后的状态轨迹。
4. 异常发生前的动作序列。
5. 候选恢复动作及其执行结果。
6. 恢复成功率、位姿误差、碰撞次数、恢复耗时等评价指标。
7. 仿真环境参数，例如摩擦系数、物体质量、接触参数、传感器噪声、物体初始位姿等。

仿真经验的作用是提供大量低成本的异常恢复候选策略，帮助系统在真机异常发生时快速获得初始恢复方案。

### 4.2 真机异常处理经验库

真机异常处理经验库主要记录真实机器人在执行任务过程中遇到的异常及其处理结果。相比仿真经验，真机经验数量较少、采集成本较高，但它直接反映真实物理环境中的动力学、感知误差和硬件状态，因此对弥合 Sim-to-Real Gap 更有价值。

真机经验可包含以下内容：

1. 真实任务场景和目标。
2. 真机观测到的异常状态。
3. RGB-D 图像、点云、关节角、末端执行器位姿、力/力矩、夹爪状态等传感器数据。
4. 异常发生前后的真实轨迹。
5. 实际执行过的恢复动作。
6. 恢复结果，例如成功、失败、部分成功或造成新的异常。
7. 真机执行结果与仿真预测之间的偏差。

真机经验的作用不是简单替代仿真经验，而是用于校正仿真经验，使沙盒推演更接近真实执行结果。

### 4.3 Sim-Real 差异经验

本研究的关键不只是分别保存仿真经验和真机经验，而是显式保存二者之间的差异。对于相似异常场景，系统需要记录仿真预测结果与真机执行结果之间的偏差，这一差异可以被称为 Sim-Real 差异签名。

差异签名可以包含：

1. 仿真与真机之间的末端位姿误差。
2. 物体实际位移与仿真预测位移之间的差异。
3. 接触状态差异，例如仿真中成功抓取但真机中发生滑落。
4. 动作执行误差，例如关节响应延迟、夹爪闭合不足。
5. 环境参数差异，例如摩擦、质量、碰撞响应或传感器噪声。
6. 恢复策略成功概率在仿真和真机之间的差异。

该差异签名是双源经验库区别于普通经验回放库的核心。经验库不仅存储“遇到异常后应该怎么做”，还存储“仿真和现实在这个异常上差在哪里”。

### 4.4 经验写入门控

现有研究表明，长期机器人经验不能无限制保存，否则会带来存储压力、检索噪声和无效经验干扰。因此，本方案需要增加经验写入门控机制，判断一段轨迹是否值得进入长期经验库。

建议采用三类写入条件：

1. **异常触发写入**：发生抓取失败、滑落、碰撞、放置偏移、视觉遮挡等异常时，自动保存异常前后一段时间窗口。
2. **失败触发写入**：当恢复动作失败或导致新的异常时，保存失败轨迹，并标记为高风险经验。
3. **高差异触发写入**：当沙盒预测结果与真机执行结果差异较大时，保存该案例，用于后续修正 Sim-Real 差异模型。

在后续实现中，可以先不使用复杂的贝叶斯惊喜模型，而采用工程上更容易落地的门控指标：

```text
write_score =
  w1 * anomaly_score
  + w2 * failure_score
  + w3 * sim_real_gap_score
  + w4 * recovery_utility_score
```

当 `write_score` 超过阈值时，该片段进入长期经验库；否则只保存在短期缓存中。

### 4.5 经验类型划分

结合机器人记忆相关论文，经验库可以进一步划分为四类经验：

1. **时间经验**：异常发生的时间顺序、动作序列、失败前兆和恢复过程。
2. **空间经验**：物体位置、障碍物位置、末端执行器姿态、接触关系和场景布局。
3. **对象经验**：不同物体的尺寸、材质、摩擦、易滑落程度和抓取稳定性。
4. **程序性经验**：可复用的恢复动作模板，例如重新对准、二次抓取、避障重规划、放置姿态修正等。

这四类经验可以共用统一的经验编号，但在检索时使用不同特征，以避免把语义相似但物理状态不相似的经验错误召回。

## 5. 经验数据结构

每条经验可以设计为统一的数据结构：

```text
Experience {
  id: 唯一编号
  source: sim / real
  task_type: grasp / place / push / insert / other
  anomaly_type: grasp_miss / object_slip / collision / pose_shift / occlusion / other
  pre_state: 异常发生前状态
  anomaly_state: 异常发生时状态
  action_before_anomaly: 异常发生前动作
  recovery_action: 恢复动作
  outcome: success / fail / partial_success
  metrics: {
    success_rate,
    pose_error,
    collision_count,
    recovery_time,
    safety_score
  }
  sim_params: {
    friction,
    mass,
    contact_params,
    sensor_noise,
    object_pose
  }
  real_feedback: {
    observed_pose_error,
    force_torque_feedback,
    gripper_feedback,
    execution_delay
  }
  residual_signature: {
    sim_real_pose_gap,
    sim_real_contact_gap,
    sim_real_outcome_gap,
    estimated_param_gap
  }
  retrieval_keys: {
    visual_embedding,
    state_action_embedding,
    object_id,
    anomaly_label,
    task_stage
  }
  critic_result: {
    collision_risk,
    joint_limit_risk,
    gripper_risk,
    predicted_success
  }
}
```

在实际实现中，可以根据实验平台先保留必要字段，避免一开始设计过于复杂。

对于最小可实现版本，建议优先保留以下字段：

1. `source`
2. `task_type`
3. `anomaly_type`
4. `anomaly_state`
5. `recovery_action`
6. `outcome`
7. `metrics`
8. `sim_params`
9. `real_feedback`
10. `residual_signature`

其他字段可在系统稳定后逐步扩展。

## 6. 经验检索与候选恢复生成

经验检索不能只依赖自然语言相似度，因为机器人异常恢复强依赖物理状态。建议采用混合检索：

1. **语义检索**：根据任务类型、异常类型和对象类别检索候选经验。
2. **状态检索**：根据机器人关节状态、末端位姿、物体位姿和夹爪状态检索物理相似经验。
3. **轨迹检索**：根据异常前后的状态-动作序列检索相似失败轨迹或恢复轨迹。
4. **差异检索**：优先召回历史上 Sim-Real 差异模式相似的经验。

候选恢复动作可以来自三类来源：

1. 历史经验中的成功恢复动作。
2. 规则化恢复模板，例如重新对准、后退、抬升、重新抓取、放置点修正。
3. 规划器或策略模型生成的新动作。

检索后的候选动作不应直接执行，而是进入沙盒推演和 critic 评分阶段。

## 7. 沙盒推演机制

沙盒推演机制是本研究的决策核心。当真机检测到异常后，系统首先将当前状态映射到仿真环境中，构建一个与真实异常现场尽可能一致的数字副本。随后，系统从经验库中检索相似异常案例，并生成若干候选恢复动作。

沙盒推演主要包括以下步骤：

1. **异常状态同步**：将真机当前的物体位姿、机器人状态和环境信息同步到仿真环境。
2. **经验检索**：根据当前异常状态检索相似的仿真经验和真机经验。
3. **差异估计**：利用真机经验中的 Sim-Real 差异签名修正仿真参数或候选策略。
4. **候选动作生成**：从历史成功恢复动作、规则策略或规划器中生成多个恢复动作。
5. **仿真推演验证**：在沙盒中执行候选动作，评估成功率、碰撞风险和任务完成质量。
6. **策略选择**：选择综合评分最高的恢复动作。
7. **真机执行与回写**：将恢复动作下发至真机，并将执行结果写回经验库。

该机制使真机具备一种“先模拟、后执行”的异常处理能力。

沙盒推演阶段需要引入轨迹评分函数，而不是只判断成功或失败。建议评分函数包含：

```text
sandbox_score =
  a1 * predicted_success
  - a2 * collision_risk
  - a3 * pose_error
  - a4 * recovery_time
  - a5 * sim_real_uncertainty
```

其中 `sim_real_uncertainty` 表示当前候选动作在历史经验中是否存在较大的仿真-真机不一致。如果某个动作在仿真中成功，但历史上类似动作在真机中经常失败，则需要降低其评分。

## 8. Sim-Real 差异更新机制

真机执行完成后，系统需要比较沙盒预测与真实执行结果，并更新 Sim-Real 差异经验。

可以定义：

```text
sim_real_gap =
  b1 * pose_gap
  + b2 * contact_gap
  + b3 * outcome_gap
  + b4 * execution_delay_gap
```

其中：

1. `pose_gap` 表示仿真预测末端位姿或物体位姿与真机观测之间的误差。
2. `contact_gap` 表示接触状态差异，例如仿真中稳定抓取但真机中发生滑落。
3. `outcome_gap` 表示仿真成功但真机失败，或仿真失败但真机成功。
4. `execution_delay_gap` 表示执行延迟、控制响应和夹爪闭合状态差异。

如果 `sim_real_gap` 较小，说明当前沙盒对真实环境有较好预测能力；如果较大，则说明该类异常场景需要被重点写入经验库，并在后续检索中作为差异校正案例使用。

## 9. 初步实验设计

为了保证研究可以落地，初期实验不宜设计得过大。建议选择长时程 pick-and-place 任务中的异常恢复作为实验场景。

可选异常包括：

1. 抓取失败：夹爪未能正确抓住目标物体。
2. 物体滑落：抓取后物体在移动过程中滑落。
3. 放置偏移：目标放置位置发生偏移。
4. 碰撞异常：机械臂或夹爪与障碍物发生碰撞。
5. 视觉遮挡：目标物体被遮挡导致定位误差。

可以设置以下对比方法：

1. **无经验库方法**：异常发生后直接使用固定恢复策略。
2. **仅仿真经验库方法**：只检索仿真异常经验并执行恢复策略。
3. **仅真机经验库方法**：只利用历史真机异常经验。
4. **双源经验库方法**：同时利用仿真经验、真机经验和 Sim-Real 差异进行沙盒推演。

评价指标包括：

1. 异常恢复成功率。
2. 平均恢复时间。
3. 真机试错次数。
4. 碰撞次数或安全风险。
5. 最终任务完成率。
6. 仿真预测结果与真机执行结果的一致性。

如果双源经验库方法在恢复成功率、试错次数和安全性方面优于其他方法，就可以验证本研究的核心假设。

为了更贴合现有论文和实验可行性，建议第一阶段只选择两类异常：

1. **抓取失败或抓取不稳**：对应接触、夹爪闭合和物体位姿误差，容易体现 Sim-Real Gap。
2. **放置偏移或堆叠失败**：对应长时程 pick-and-place 的后半段异常，容易体现恢复策略对最终任务完成率的影响。

第一阶段不建议同时覆盖导航、复杂遮挡和大规模开放世界任务，否则实验变量过多，难以证明双源经验库本身的贡献。

## 10. 预期创新点

本研究可以形成以下创新点：

1. 提出一种面向机器人异常恢复的 Sim-Real 双源经验库结构，同时保存仿真异常经验、真机异常经验和二者之间的差异信息。
2. 提出一种面向异常恢复的经验写入门控机制，优先保存异常、高失败风险和高 Sim-Real 差异片段。
3. 提出一种基于混合检索和差异校正的沙盒推演机制，使真机在执行恢复动作前能够先进行仿真验证。
4. 引入候选恢复动作的沙盒评分与 critic 检查，降低碰撞、关节越界和错误抓取等风险。
5. 将异常恢复从直接真机试错转化为“经验筛选-混合检索-沙盒推演-真机执行-差异回写”的闭环过程。
6. 在长时程 pick-and-place 异常恢复任务中验证双源经验库对 Sim-to-Real Gap 弥合和异常恢复成功率提升的作用。

## 11. 当前方案需要进一步明确的问题

后续需要继续细化以下问题：

1. 经验库的最小可实现字段是什么。
2. 当前实验平台能够采集哪些真机状态和传感器数据。
3. 异常检测是使用规则、视觉模型，还是轨迹偏差检测。
4. 相似经验检索采用向量检索、轨迹距离，还是混合检索。
5. Sim-Real 差异签名如何计算。
6. 沙盒推演使用 MuJoCo、Isaac Sim，还是其他仿真平台。
7. 恢复动作来自规则库、轨迹库、规划器，还是学习策略。
8. 实验中优先验证哪一种异常类型。

## 12. 阶段性结论

当前研究方案的核心价值在于，它不是单纯构建一个机器人经验存储系统，而是利用双源经验库服务于真实机器人的异常恢复决策。仿真经验提供大规模、低成本的恢复候选；真机经验提供真实物理反馈；Sim-Real 差异签名用于修正仿真沙盒；沙盒推演则用于降低真机直接试错的风险。

因此，本研究可以被凝练为：

**构建一个面向异常恢复的 Sim-Real 双源经验库，使真实机器人在异常处理前能够基于历史经验进行仿真沙盒推演，从而提升真实环境中的恢复成功率并降低试错风险。**
