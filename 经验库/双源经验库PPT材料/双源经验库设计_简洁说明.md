# 双源经验库设计简洁说明

## 1. 设计目标

双源经验库不是简单保存历史日志，而是把一次机器人异常恢复 episode 组织成可检索、可验证、可比较的结构化经验。设计目标包括三点：保留仿真经验的可重复性，接入真机/伪真机经验，并在在线策略中区分支持证据和风险证据。

## 2. 总体结构

系统包含两类经验来源：仿真经验和真机/伪真机经验。两类经验最终都写入 MemoryV3+，形成统一经验条目。经验条目再经过 sim-real pair/gap 计算，生成可信度和风险信号，最后用于检索排序、候选方案评分和 LLM prompt 提示。

核心链路是：

```text
仿真经验 + 真机/伪真机经验
        ↓
MemoryV3+ 统一经验条目
        ↓
Sim-Real Gap 与可信度校准
        ↓
Gap/Risk-aware 检索、Prompt 提示、Candidate Scoring
```

## 3. MemoryV3+ 条目保存什么

一条经验包含任务与异常、动作与结果、证据与传感器、双源与风险四类信息。关键字段包括：

- `condition_id`、`scenario_id`、`task_stage`
- `skill_sequence`、`recovery_plan`
- `result`、`execution_feedback`、`failure_reason`
- `keyframes`、`key_slices`
- `sensor_summary`
- `real_episode_ref`、`raw_refs`
- `sim_real_pair`、`sim_real_gap`
- `critic_result`、`memory_gate`

其中 `sensor_summary` 保存压缩后的关节、末端、夹爪、接触、力觉和时间戳摘要；高频原始数据通过 HDF5、视频目录或机器人日志路径外部引用。

## 4. 经验如何获取

仿真经验由实验运行自动写入。`ExperimentV4._build_experience_entry()` 会记录异常条件、感知结果、恢复动作、关键帧、接触状态、物体高度、pinch distance 和仿真传感器摘要。

真机/伪真机经验由两层代码处理：

- `real_memory/collector.py`：从真机 episode 目录收集 `episode.json`、`video/`、`keyframes/`、`robot_log.jsonl`、`episode.hdf5` 等产物。
- `real_memory/ingest.py`：将标准 real episode JSON 转换成 MemoryV3+ 条目。

如果没有手写 `sensor_summary.json`，`real_memory/log_parsers.py` 会自动从 JSONL、JSON samples 或 HDF5 中提取关节位置、关节速度、末端位姿、夹爪状态、接触状态和时间戳。

## 5. Sim-Real Gap 如何工作

系统将可比较的仿真经验和真机/伪真机经验配对，计算可解释 gap：

- `outcome_gap`：仿真和真实成败是否一致
- `pose_gap`：物体观测位置和恢复后高度差异
- `contact_gap`：闭合/抬升接触模式差异
- `perception_gap`：感知位置差异
- `actuation_gap`：夹爪闭合和 pinch distance 差异
- `robot_state_gap`：关节位置和夹爪状态差异

如果出现 `sim_success_real_fail`，说明该方案在仿真中成功但在真机/伪真机中失败，后续检索和 prompt 中会作为风险证据。

## 6. 在线策略如何使用经验

在线检索使用 `gap_aware=True` 和 `risk_aware=True`。系统会提高真实成功经验权重，降低失败、高 gap、高 critic 风险经验的排序。

当前 scoring 主要包含：

- `real_success_bonus`：真实/伪真机成功经验加权
- `paired_bonus`：已完成 sim-real 配对的经验少量加权
- `sim_real_failure_penalty`：仿真成功但真实失败经验显著降权
- `gap_score_penalty`：gap 大的经验降权
- `gap_uncertainty_penalty`：不确定性高的经验降权
- `critic_penalty` 和 `failure_penalty`：critic 风险或失败经验降权

LLM prompt 中也会显示双源提示，例如经验来源、验证状态、Sim-Real Gap、是否存在“仿真成功但真实失败”的风险。

## 7. 已实现与后续边界

已经实现：MemoryV3+ schema、仿真经验写入、真机/伪真机导入、原始日志解析器、sim-real gap、gap/risk-aware scoring、prompt 风险提示和 smoke 验证。

仍需后续接入：ROS/ROS2 或厂商 SDK 在线采集 adapter，真实机器人批量验证实验，基于真实数据标定 robot_state_gap 权重，以及数据驱动 gap critic。
