# 论文方法章节草稿：面向机器人异常恢复的 Sim-Real 双源经验库与沙盒推演

## 1. 方法概述

本文提出一种面向机器人异常恢复的 Sim-Real 双源经验库方法。不同于仅保存仿真轨迹或仅使用真实成功案例的经验记忆，本方法将仿真执行经验、真实/伪真实执行经验、二者之间的差异、critic 风险评估和沙盒校准信息统一到一个可检索的结构化经验库中。

当机器人在执行任务时遇到异常，系统并不直接执行由语言模型生成的恢复动作，而是先完成以下过程：

```text
异常状态编码
→ 双源经验检索
→ sim-real gap 与失败风险分析
→ 候选恢复策略生成
→ 校准后的沙盒推演
→ critic 安全评估
→ 候选策略重写或执行
→ 执行结果回写经验库
```

该方法的核心思想是：仿真经验提供低成本的恢复先验，真实经验提供实际可执行性证据，sim-real gap 则显式刻画“仿真预测为什么不能直接信任”。因此，系统不仅学习什么策略曾经成功，也学习哪些仿真成功在真实执行中可能失败。

## 2. 双源经验表示

每条经验表示为结构化条目：

```text
e = {
  task,
  anomaly,
  perception,
  robot_state,
  skill_sequence,
  execution_feedback,
  result,
  source,
  validation_status,
  memory_gate,
  critic_result,
  sim_real_pair,
  sim_real_gap,
  sandbox_calibration,
  real_episode_ref,
  memory_tags,
  retrieval_key
}
```

其中 `source` 用于区分经验来源：

```text
simulation
real
pseudo_real
human_label
```

`validation_status` 用于区分验证强度：

```text
failed
simulation_validated
shadow_validated
real_executed
real_validated
```

本方法不把真实经验简单作为更高权重的成功样本，而是进一步记录仿真与真实之间的配对关系和差异签名。

## 3. 经验写入门控

长期经验库不应保存所有轨迹。大量正常中间帧会稀释检索质量，并增加后续 planning prompt 的噪声。因此系统引入经验写入门控：

```text
write_score =
  w_a anomaly_score
  + w_f failure_or_boundary_score
  + w_g sim_real_gap_score
  + w_c critic_warning_score
  + w_u recovery_utility_score
```

当经验满足以下条件时，更可能被写入长期库：

1. 包含异常发生或恢复关键点。
2. 恢复成功或失败边界清晰。
3. 仿真与真实之间存在显著差异。
4. critic 给出 warning 或 reject。
5. 对未来相似异常具有可复用价值。

本实现的第一版采用非破坏性门控：系统记录 `memory_gate`，但默认不丢弃经验。这使得早期实验能够保留完整证据，并在数据量扩大后再切换到强制门控。

## 4. Sim-Real 经验配对

给定仿真经验集合 `E_sim` 和真实/伪真实经验集合 `E_real`，系统根据以下条件寻找 pair：

```text
same scenario_id
same condition_id
similar task_stage
similar skill_sequence signature
compatible available_actions
```

配对结果表示为：

```text
sim_real_pair = {
  pair_id,
  sim_experience_id,
  real_experience_id,
  paired_by,
  pair_score,
  gap_score,
  validation_status
}
```

该 pair 不是最终结论，而是 gap 计算的证据绑定。pair_score 低的样本可以进入人工复核或弱证据池。

## 5. Sim-Real Gap 建模

对每个 sim-real pair，系统计算差异签名：

```text
sim_real_gap = {
  gap_id,
  gap_score,
  pose_gap,
  contact_gap,
  outcome_gap,
  perception_gap,
  actuation_gap,
  scene_reconstruction_gap,
  uncertainty,
  evidence
}
```

其中：

```text
pose_gap:
  object_pose_error
  z_after_recovery_gap

contact_gap:
  contact_mismatch
  sim_contact_pattern
  real_contact_pattern

outcome_gap:
  matched_success
  sim_success_real_fail
  sim_fail_real_success
  matched_failure

perception_gap:
  pose_estimation_gap
  sim_perceived_pos
  real_perceived_pos

actuation_gap:
  gripper_closure_gap
  sim_pinch_distance
  real_pinch_distance
```

`sim_success_real_fail` 是最关键的差异类型。它表示某个策略在仿真中通过，但在真实或伪真实执行中失败。该类经验不会被当作普通失败样本丢弃，而是用于降低相似候选策略的真实可执行性评分。

## 6. 结构化 Critic

系统引入结构化 critic 表示，用于保存沙盒推演或真实执行后的风险评估：

```text
critic_result = {
  overall_status,
  critic_risk_score,
  rule_flags,
  llm_flags,
  feedback_for_rewrite,
  evidence
}
```

第一版 critic 使用规则实现，覆盖以下情况：

1. 虚拟验证失败。
2. 抓取闭合后无接触。
3. 提升阶段接触丢失。
4. 执行结果和预期恢复目标不一致。

critic 的作用不是替代恢复规划，而是为候选策略排序和重写提供约束。当 critic 风险较高时，系统可触发 LLM 重新生成局部恢复步骤。

## 7. Gap-Aware 与 Risk-Aware 检索

传统经验检索通常根据任务、异常类型、动作序列相似度返回 top-k 经验。本方法在基础相似度上加入双源修正：

```text
score(e) =
  base_similarity(e, q)
  + real_validation_bonus(e)
  + trust_bonus(e)
  - gap_uncertainty_penalty(e)
  - critic_risk_penalty(e)
  - sim_real_failure_penalty(e)
```

其中：

1. `real_validation_bonus` 提升真实执行或真实验证经验权重。
2. `trust_bonus` 提升 matched real success 经验。
3. `gap_uncertainty_penalty` 降低高不确定 gap 经验。
4. `critic_risk_penalty` 降低 critic 风险经验。
5. `sim_real_failure_penalty` 降低仿真成功但真实失败的经验。

这样，系统可以避免把“仿真中成功但真实中失败”的经验误当作高可信成功案例。

## 8. 候选恢复策略评分

给定候选动作序列 `π` 和检索经验集合 `R`，系统计算候选评分：

```text
candidate_score(π, R) =
  α support_score
  + β real_success_prior
  - γ failure_overlap_risk
  - δ gap_uncertainty
  - η critic_risk
```

输出：

```text
candidate_score
decision
support_score
real_success_prior
failure_overlap_risk
gap_uncertainty
critic_risk
evidence
```

决策规则：

```text
prefer:
  候选有真实成功先验或低风险支持。

allow:
  候选证据不足但没有明显风险。

rewrite_recommended:
  候选与历史失败或高 gap 经验重叠，应重写。

reject_recommended:
  候选存在明显高风险，不建议执行。
```

该评分机制使失败经验和 gap memory 成为规划约束，而不是简单的负样本。

## 9. 沙盒校准

真实/伪真实经验中的 gap 可用于校准下一次沙盒推演。系统根据检索到的 gap memory 估计：

```text
object_pose_bias
gripper_delay_bias
slip_risk_bias
contact_success_bias
perception_noise_bias
calibration_confidence
```

校准后的沙盒用于：

1. 初始化更接近真实异常状态的物体位置。
2. 调整抓取 attach 判定距离。
3. 记录推演时的风险偏置。
4. 影响候选策略评分。

第一版 calibration 是规则加权平均。后续可以替换为基于真实数据学习的 residual model，但不需要改变经验 schema。

## 10. 执行闭环

完整闭环如下：

```text
1. 异常发生，记录 anomaly_state。
2. 从经验库检索 success memory、failure memory、gap memory、critic cases。
3. LLM 或策略模块生成候选恢复步骤。
4. 使用 score_candidate_plan 对候选排序。
5. 从 gap memory 生成 sandbox_calibration。
6. 在校准沙盒中推演候选策略。
7. critic_result 判断是否通过、重写或拒绝。
8. 真机或伪真实执行。
9. 将执行结果导入 real memory。
10. 与 simulation memory 配对，更新 sim_real_gap。
```

该闭环将真实执行结果显式反馈给仿真沙盒，使系统随着真实经验积累逐步减少重复失败。

## 11. MVP 实现与结果

当前工程实现位于 `experiment-sim-wrapper1`。

MVP 命令：

```bash
python -B tools/run_mvp_dual_source_pipeline.py \
  --config configs/mvp_dual_source_gap_critic_u3_u4_v1.json
```

当前结果：

```text
real_entry_count = 8
sim_entry_count = 8
pair_count = 8
```

U4-3 专项结果：

```text
real_entry_count = 40
sim_entry_count = 40
pair_count = 40
matched_success = 20
sim_success_real_fail = 20
```

在 U3-4 上，系统检索到仿真成功但真实失败的 gap memory：

```text
candidate_score = 0.4148
decision = rewrite_recommended
outcome_gap = sim_success_real_fail
```

在 U4-2 上，系统检索到真实成功先验和部分失败证据：

```text
candidate_score = 0.7216
decision = prefer
```

在 U4-3 上，系统同时检索支持证据和风险证据：

```text
candidate_score = 0.5292
decision = allow
failure_overlap_risk = 0.8571
gap_uncertainty = 0.7286
```

该结果表明，普通 top-k 检索容易只返回成功样本并给出过度乐观判断；加入失败风险分支后，系统能够显式考虑与候选动作高度重叠的失败经验，从而给出更审慎的执行前判断。

该结果说明：双源 gap 能将仿真成功经验和真实可执行性区分开，并影响候选恢复策略决策。

## 12. 与相关工作的关系

本方法吸收了多类已有工作的启发，但工程落点不同：

1. 长期记忆工作说明经验需要选择性写入，因此引入 `memory_gate`。
2. 失败学习工作说明失败经验是风险先验，因此引入 `failure_overlap_risk`。
3. 沙盒规划工作说明执行前推演可降低真实试错，因此接入 `build_virtual_scene()`。
4. 轨迹 critic 工作说明成功不等于安全，因此引入 `critic_result`。
5. RAP 类检索增强规划说明经验应按阶段动态检索，因此检索键包含 task stage、gap、critic、memory role。
6. 机器人记忆评测工作说明记忆应分层和多类型，因此加入 `memory_tags` 和 `sim_real_gap_memory`。

本文区别于已有工作的关键点在于：将 sim-real difference 本身经验化，并将其用于沙盒校准和候选策略风险评估。

## 13. 局限性

当前方法仍有以下限制：

1. MVP 使用 pseudo-real 数据，还不是完整真机闭环。
2. pair/gap 规则依赖 result.json 中已有字段，真实数据需要更严格的同步和标注。
3. critic 目前是规则化实现，尚未训练视觉/轨迹 critic。
4. calibration 是规则加权平均，无法覆盖复杂接触动力学差异。
5. 视觉关键帧尚未进入 embedding 检索。

这些限制不影响第一版方法成立，但决定了后续实验必须补充真实数据、批量对照和消融。

## 14. 实验设计建议

最终论文实验应至少包含以下方法：

| 方法 | Sim memory | Real memory | Gap | Critic | Calibration |
| --- | --- | --- | --- | --- | --- |
| Direct LLM | 否 | 否 | 否 | 否 | 否 |
| Sim-only | 是 | 否 | 否 | 否 | 否 |
| Sim memory | 是 | 否 | 否 | 否 | 否 |
| Hierarchical memory | 是 | 否 | 否 | 部分 | 否 |
| Dual-source gap memory | 是 | 是 | 是 | 否 | 是 |
| Dual-source gap critic | 是 | 是 | 是 | 是 | 是 |

核心指标：

```text
recovery_success_rate
unsafe_motion_count
repeated_failure_rate
sim_real_prediction_error
critic_rewrite_count
critic_rejection_count
memory_usefulness
```

推荐消融：

1. 去掉 real memory。
2. 去掉 sim-real gap。
3. 去掉 critic。
4. 去掉 sandbox calibration。
5. 只使用失败经验，不使用成功先验。
6. 只使用成功经验，不使用失败风险。

## 15. 小结

本文方法将仿真经验、真实经验和二者差异统一为可检索、可评分、可校准的异常恢复经验库。系统不是简单地扩大记忆容量，而是将 sim-real gap 作为核心知识单元，用于识别仿真成功在真实环境中可能失败的情况。

当前 MVP 已经证明 schema、导入、配对、gap、calibration、retrieval 和 candidate scoring 链路可行。下一阶段需要将双源方法接入完整在线 batch，并用真实或更高保真的 pseudo-real 数据验证其对恢复成功率、重复失败率和安全风险的改善。
